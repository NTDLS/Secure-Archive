Configuration - Advanced



This tab gives you a detailed list of your installed processors and their capabilities. You can use this setup tab to tell Secure Archive which processors it is allowed to use.

See below the image for a detailed break down of options.



Pre-allocate disk space for new archives::

  • This option instructs secure archive to pre-allocate disk space for any operation that would increase the size of the archive you are working on. The program will allocate a "worse case scenario" amount of disk space, but will release the unused amount after the processes completes.

Use copy process instead of move process on drag drop:
  • This option changes the behavior of the windows drag-drop file operation after you drag a file from secure archive. If you are dragging a file to a disk other than the disk specified in your "temporary storage location" (below), then the copy process if more secure. However, the move process is generally faster.

Secure Delete Passes:
  • The number of time Secure Archive will overwrite any temporarily created files. This is also the default number of passes that will be used on any other delete process with any of Secure Archives windows.

Temporary storage location:
  • The full disk and path that Secure Archive should use for temporary storage (if any is required). If this box is left blank, then Secure Archive will use your default profile temp directory.
